# Project Management Workflow for Automated Context Management

## Introduction
This document outlines a comprehensive project management workflow designed for effective automated context management, feature tracking, systematic questioning, and recap generation before launching project phases. 

## Objectives
1. **Automated Context Management**
   - Maintain up-to-date records of project details, decisions, and changes.
   - Ensure easy access to current context for all involved stakeholders.

2. **Feature Tracking with UTC Timestamps**
   - Utilize UTC timestamps in ISO 8601 format to log all relevant project events.
   - Ensure consistency and clarity in recording project phases, tasks, and feature updates.

3. **Systematic Questioning of Missing Details**
   - Before launching any phase, conduct a thorough review to identify missing information.
   - Collaborate with the team to fill in any gaps to ensure thorough understanding and coverage.

4. **Recap Generation Before Phase Launch**
   - Generate a project recap document summarizing key points, decisions, and tasks before the start of each project phase.
   - Distribute the recap to all stakeholders to ensure collective understanding and readiness.

## Workflow Steps
1. **Initiation Phase**
   - Create a project outline and task list.
   - Define the project scope and objectives.
   - Document the initial details and context.

2. **Feature Development Tracking**
   - Whenever a new feature is started or updated, log the following information with a timestamp:
     - Feature Name
     - Description 
     - Developer Assigned
     - Status (In Progress / Completed)
     - UTC Timestamp (ISO 8601 format)  

3. **Checkpoint Reviews**
   - Establish regular checkpoints for project updates and feature completions.
   - At each checkpoint:
     - Verify that all tasks are on track and documented.
     - Conduct systematic questioning to uncover any missing details. Document these queries and answers promptly.

4. **Recap Generation**
   - Before launching a new phase:
     - Compile a recap of completed tasks and decisions made during the previous phase.
     - Ensure recaps include feedback from team members and any outstanding issues.
   - Distribute the recap to all team members to ensure alignment. 

5. **Phase Launch**
   - Confirm all necessary details are in place for the next phase.
   - Schedule a kickoff meeting with all stakeholders to discuss the recap and the agenda moving forward.
   
## Conclusion
The outlined workflow serves as a structured approach to project management, focusing on context clarity, thorough tracking, systematic questioning, and the importance of communication through recaps before any project phase launches. 
This framework is designed to foster collaboration and ensure all necessary details are captured effectively throughout the project's lifecycle.
